# shopping-ai プロダクト構想

## サービス概要
ショッピングサイトの商品をAIが分析し、購入をサポートするWebサービス。

## MVPで実装済みの機能
- トップページ
- 商品URL入力フォーム
- AI解析結果ページ(現在はダミーデータ)
- スマホ対応デザイン
- GitHub + Vercelでの公開

## 今後実装したい主な機能
- ユーザーが入力した商品URLから、実際の商品情報を取得する
- AIがレビューを分析・要約する
- 「良い点」「悪い点」「おすすめな人」「おすすめしない人」を表示する
- AIが100点満点で商品を評価する
- 類似商品を提案する
- 複数のショッピングサイト(Amazon、楽天市場、Yahoo!ショッピング)に対応する

## 将来的な機能(優先度は低くて良い)
- 価格比較・価格推移
- 買い時判定
- お気に入り登録・ログイン機能
- 価格通知
- AIチャットでの商品相談
- 広告・アフィリエイトによる収益化

## SEO方針(重要)
- 商品が初めて解析されたら、その商品専用のページを自動生成してDBに保存する
- 同じ商品は再度AI解析せず、保存済みデータを再利用してAPIコストを削減する
- 生成したページはGoogleにインデックスされることを狙う

## 技術スタック
Next.js / TypeScript / React / Supabase / GitHub / Vercel / AI API(Gemini Flash候補)

## 開発ルール(必ず守ってください)
- 一度に大量のコードを書かず、小さな単位で実装する
- 実装の前に「何をするか」を簡潔に説明する
- 保守しやすいファイル構成にする
- 必要なライブラリは理由も説明する
- エラーが出たら原因と修正方法も説明する
- ユーザーがそのまま実行できるコマンドを教える
- セキュリティやパフォーマンスも考慮する
- より良い実装方法があれば提案する
- ユーザーはプログラミング初心者なので、専門用語には簡単な説明を添える
