以下を設定すると、これらが本来の姿(実際のAI解析・キャッシュ・SEO用サイトマップ)で動くようになります。

## 1. Supabaseのセットアップ(DB保存・キャッシュ・類似商品・サイトマップに必要)

1. https://supabase.com でアカウント作成 → 新規プロジェクトを作成
2. プロジェクトの **SQL Editor** を開き、リポジトリ内の `supabase/schema.sql` の中身を貼り付けて実行(`products` テーブルが作成されます)
3. プロジェクトの **Settings → API** を開き、以下の2つをコピー
   - `Project URL` → `SUPABASE_URL`
   - `service_role` キー(**anonキーではありません**。secretと書かれている方) → `SUPABASE_SERVICE_ROLE_KEY`
4. ローカルで試す場合: リポジトリ直下に `.env.local` を作成し、`.env.example` を参考に値を貼り付ける
5. 本番(Vercel)に反映する場合: Vercelのプロジェクト → **Settings → Environment Variables** で
   `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` を追加 → 再デプロイ(Redeploy)

   ```
   # ローカルで動作確認したい場合、リポジトリ直下で実行
   cp .env.example .env.local
   # その後 .env.local を開いて値を入力し、以下で確認
   npm run dev
   ```

> `SUPABASE_SERVICE_ROLE_KEY` は強い権限を持つ秘密鍵です。GitHubには絶対にコミットしないでください(`.gitignore`で`.env*`は除外済みです)。

## 2. Google AI Studio (Gemini) のセットアップ(AI解析に必要)

1. https://aistudio.google.com/apikey でGoogleアカウントでログインし、APIキーを発行
2. `GEMINI_API_KEY` としてローカルの `.env.local` と、Vercelの Environment Variables に追加 → 再デプロイ

設定すると、商品ページの「良い点」「悪い点」「おすすめな人」などがAIによる実際の生成結果に変わります。

## 3. (任意・後回しでOK) 独自ドメインを設定したとき

独自ドメインをVercelで設定したら、`NEXT_PUBLIC_SITE_URL`(例: `https://your-domain.com`)を
環境変数に追加してください。設定しなくても、Vercelが自動発行するURLでSEO用のURLは正しく生成されます。

### Phase3: AIにレビュー本文をまだ渡していません

「AIがレビューを分析・要約する」という構想に対し、今回はまだ**実際のレビュー本文を集める仕組みを作っていません**
(レビュー欄は商品ページよりもさらにスクレイピングが難しく、ページ送りやログインが必要なことが多いためです)。

暫定案として、商品名・価格・販売サイトの情報だけをAIに渡し、「個別の欠陥/長所を断定しない」よう
指示を入れることで、もっともらしい嘘(ハルシネーション)を避けています。ただしこれは
**実際のレビューを読んで要約しているわけではありません**。

次に進める場合、2つの方向性があります。判断をお願いします。

- (A) このまま: 商品名・価格から一般的な観点でAI解析を続ける(実装コストは低いが、レビュー分析としては弱い)
- (B) レビュー本文の取得に本格的に取り組む(各サイトのレビュー欄をスクレイピング、または口コミAPIの利用を検討。実装コストは高いが構想により近づく)

戻られたら「Bで進めて」などと指示いただければ、そこから着手します。

## 既知の制約: Amazonの自動アクセス制限

Amazonは、通常のスクレイピング(ブラウザのふりをしたHTTPリクエスト)に対してCAPTCHAページを
返してブロックすることが多いです。今回はCAPTCHA回避のような技術(自動化検知の突破)は
意図的に実装していません(利用規約違反・倫理的に問題があるため)。

現状、Amazon商品を解析しようとすると「Amazonは自動アクセスを制限している場合があります」という
エラーメッセージが表示される可能性が高いです(楽天・Yahoo!は問題なく動作します)。

Amazonにも確実に対応したい場合、以下のような選択肢があります(いずれもコスト・アカウント登録が必要):

- Amazon Product Advertising API(公式API)の利用 → Amazonアソシエイト(アフィリエイト)登録が必要
- 有料のスクレイピング代行サービス(ScraperAPIなど)の利用 → 別途契約・費用が発生

これも急ぎでなければ、まずは楽天・Yahoo!ショッピングを中心に運用し、必要になったタイミングで
相談いただく形で良いと考えています。
